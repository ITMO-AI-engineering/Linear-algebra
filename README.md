# Лабораторная работа 1: Реализация класса Matrix с разреженным хранением (CRS)

## Описание
Реализация класса Matrix для работы с разреженными матрицами в формате CRS (Compressed Row Storage). Класс предоставляет функциональность для эффективного хранения и обработки разреженных матриц, где большинство элементов равны нулю.

## Основные возможности

### Инициализация и хранение
- **Разреженное хранение**: матрица хранится в формате CRS для экономии памяти
- **Три основных массива**:
  - `value` - значения ненулевых элементов
  - `cols` - номера столбцов ненулевых элементов
  - `rows` - указатели на начало строк

### Операции с матрицами
1. **Сложение матриц** (`__add__`)
   - Поэлементное сложение двух матриц одинакового размера
   - Автоматическое сохранение разреженной структуры

2. **Умножение** (`__mul__`)
   - Умножение матрицы на скаляр
   - Умножение матрицы на матрицу (с проверкой совместимости размеров)

3. **Вычисление определителя** (`get_determine`)
   - Рекурсивное вычисление определителя для квадратных матриц
   - Использование метода разложения по строке

4. **Вычисление следа** (`get_trace`)
   - Сумма элементов главной диагонали
   - Только для квадратных матриц

5. **Доступ к элементам** (`get_elem`)
   - Эффективный доступ к элементам с учетом разреженной структуры
   - Автоматическое возвращение 0 для отсутствующих элементов

### Обработка ошибок
- Кастомный класс исключений `MatrixError`
- Проверка корректности размеров матриц
- Валидация индексов при доступе к элементам

- Теоретическое обоснование метода
- Реализацию ключевых этапов PCA
- Визуализацию результатов

### Основные компоненты
1. **Теоретическая часть**:
   - Математическое обоснование PCA
   - Доказательство оптимальности главных компонент

2. **Практическая реализация**:
   - Класс `SparseMatrixCRS` для работы с разреженными матрицами

     - Вычисления матрицы ковариаций
     - Нахождения собственных значений и векторов

3. **Визуализация**:
   - Сравнение исходных данных и их проекции
